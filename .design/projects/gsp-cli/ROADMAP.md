# Project Roadmap

## Project: gsp-cli
**Brand:** get-shit-pretty
**Created:** 2026-03-08

> Apply the GSP brand to the CLI's core terminal output.

---

## Phase 1: Brief
**Status:** pending
**Command:** `/gsp:project-brief`
**Input:** Brand system + BRIEF.md
**Output:** `brief/` (scope, adaptations, conditionals)
**Goal:** Scope the 5 core screens — what components each uses, what adaptations are needed, gap analysis against existing output.

## Phase 2: Research
**Status:** pending
**Command:** `/gsp:project-research`
**Input:** Brief scope + brand system + BRIEF.md
**Output:** `research/` (UX patterns, competitor UX, technical research, accessibility patterns)
**Goal:** Research CLI UX patterns, terminal art techniques, ANSI color tier strategies, and accessibility in terminal environments.

## Phase 3: Design
**Status:** pending
**Command:** `/gsp:project-design`
**Input:** Research + brief + brand system + BRIEF.md
**Output:** `design/` (screen chunks + shared/)
**Goal:** Design the 5 core screens composing brand components into cohesive terminal experiences.

## Phase 4: Critique
**Status:** pending
**Command:** `/gsp:project-critique`
**Input:** All prior artifacts
**Output:** `critique/` (critique + accessibility chunks)
**Goal:** Critique screen designs for consistency, usability, and terminal accessibility.

## Phase 5: Build
**Status:** pending
**Command:** `/gsp:project-build`
**Input:** Design + brief + brand system
**Output:** `build/CODE.md` + `build/components/`
**Goal:** Implement the 5 core screens as Node.js functions using ANSI escape codes and brand tokens.

## Phase 6: Review
**Status:** pending
**Command:** `/gsp:project-review`
**Input:** Built deliverables + design intent
**Output:** `review/` (acceptance report + issues)
**Goal:** Validate implemented screens match designs — token compliance, component usage, color tier fallbacks.
