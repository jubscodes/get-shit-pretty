# Personas

> Phase: design | Project: gsp-cli | Generated: 2026-03-08

---

## Primary: Alex — The Agent-Augmented Developer

- **Role:** Full-stack developer, 3-5 years experience
- **Tools:** VS Code + Claude Code (primary), occasionally OpenCode or Codex
- **Terminal:** iTerm2 or VS Code integrated terminal, macOS, truecolor support
- **Goals:**
  - Ship projects that look intentionally designed, not "default Bootstrap"
  - Use AI agents effectively without losing creative control
  - Understand the design pipeline without needing a design background
- **Pain points:**
  - CLI tools with walls of unstyled text feel untrustworthy
  - Forgets which phase comes next after stepping away
  - Overwhelmed by too many options at once — prefers guided flows
- **Usage context:** Works in 2-3 hour sessions, often returns days later. Needs fast re-orientation. Reads terminal output quickly — scanning, not studying. Values consistency: if the tool looks polished, trusts the output is polished too.

## Secondary: Jordan — The Design-Curious Developer

- **Role:** Backend or systems developer exploring design for the first time
- **Tools:** Claude Code, unfamiliar with design terminology
- **Terminal:** Default macOS Terminal or VS Code, standard 80-column width
- **Goals:**
  - Understand what "brand identity" and "design system" mean in practice
  - Follow a guided process without feeling lost
  - Get a professional-looking result without design expertise
- **Pain points:**
  - Design jargon is intimidating ("archetype", "verbal identity", "tokens")
  - Unsure if they're doing it right — needs clear progress indicators
  - Wants the tool to feel approachable, not like a design school exam
