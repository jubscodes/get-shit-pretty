# Brand Narrative

> Phase: verbal | Brand: get-shit-pretty | Generated: 2026-03-08

---

## Origin Story

A developer-designer hybrid got tired of the gap.

Not the skills gap. The process gap. The one where AI coding tools ship features in minutes but the product still looks like it was assembled by committee. Where "design" means picking a color palette on Coolors and calling it a brand. Where the terminal handles everything except the one thing that makes a product hold together.

GSP started as a question: what if the design process lived where the code already does?

Not a plugin. Not a SaaS dashboard. A structured journey — brief to build — running in the same terminal where the product ships. The dual-diamond framework, adapted for builders who think in both code and craft.

jubs.studio was the first brand built with GSP. The tool designed the designer. That's the proof point: a complete brand identity — strategy, verbal system, visual identity, component library — generated through a CLI pipeline.

Three brands now run on the same system. Same dark-first DNA. Three distinct identities. The tool scales because the process does.

---

## Narrative Arc

### 1. Setup: The Speed Era

AI coding tools changed everything. Vibe-coding is real. A solo builder can ship a working product in a weekend. The barrier to functional software dropped to near zero.

This is good. Building got democratic. More people ship more things.

### 2. Tension: The Coherence Deficit

But products that work aren't products that last. The homepage says one thing, the docs say another, and the CLI output sounds like a different tool entirely. There's no brand — there's an accumulation of decisions nobody made intentionally.

Design is either skipped ("we'll do it later"), outsourced ("find a designer on Fiverr"), or cargo-culted ("let's use the same font as Linear"). The result is products that function but don't resonate. They ship. They don't stick.

The irony: the builders who care most about craft in their code accept incoherence everywhere else.

### 3. Resolution: The Design Journey

GSP brings design engineering into the terminal. Not a color picker. Not a logo generator. A structured journey from discovery through system — the same rigor a brand agency applies, adapted for builders who live in the CLI.

Seven phases. Each one produces decisions, not decorations. A brief captures constraints. Strategy defines positioning. Verbal identity gives the brand a voice. Visual identity gives it a face. The system ties it together.

It works because it meets builders where they are. No context switch to Figma. No handoff to a designer who doesn't understand the product. The person who writes the code writes the brand, guided by a process that prevents incoherence.

### 4. Transformation: Enduring Craft

Builders ship products with brands that hold together. Not because they became designers — because they followed a process that makes design decisions explicit.

The CLI builder who used to ship a README and a default theme now ships a brand system. The designer who used to hand off mockups now builds the thing. The design engineer who always lived in the overlap finally has a tool that validates the hybrid.

Design engineering becomes the default discipline. Not a nice-to-have. Not a specialty. The way products get made.
