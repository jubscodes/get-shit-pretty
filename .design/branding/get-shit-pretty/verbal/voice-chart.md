## Before / After Rewrites

**Before (generic dev tool):**
> Welcome to GSP! We're so excited to have you here. GSP is a powerful AI-driven platform that helps developers and designers create stunning brand identities. Let's get started on your creative journey!

**After (GSP voice):**
> GSP is a design engineering system for the terminal. It turns a blank brief into a shipped brand — strategy, verbal identity, visual system, components. Run `/gsp:brand-discover` to start.

**Before (corporate changelog):**
> We're thrilled to announce version 0.4.0, which includes exciting new features and improvements to enhance your experience.

**After (GSP voice):**
> v0.4.0 ships the dual-diamond refactor. Branding and project pipelines now run independently. New: `/gsp:doctor` for project health checks. Breaking: config format changed. See migration guide.

**Before (vague error):**

## Grammar and Style Rules

| Rule | GSP Standard | Notes |
|------|-------------|-------|
| **Contractions** | Yes | "it's," "don't," "you'll" — natural speech |
| **Oxford comma** | Yes | "strategy, verbal, and identity" |
| **Exclamation marks** | Sparingly | Never in docs or terminal. OK in celebratory success states |
| **Emoji** | Strategic only | The `✧` sparkle is ours. No `🚀🎉💪🔥` |
| **First person** | "GSP" or "we" | Never "I." GSP is a system, not a persona |
| **Second person** | "You" | Always. Direct address. No "the user" in copy |
| **Swearing** | Yes, strategically | The name sets the bar. "Shit" is fine. Escalation beyond that needs a reason |
| **Sentence length** | Short average (15 words) | Punchy. Fragments are fine. Varies for rhythm |
| **Paragraphs** | 1-3 sentences | Scannable. White space is a feature |
| **Jargon** | Use it, don't explain it | Audience knows "archetype," "positioning," "component." Define only GSP-specific terms |
| **Hedging** | Avoid | "This works" not "this should work." "Run this" not "you might want to run this" |
| **Capitalization** | Sentence case | Headlines, buttons, labels — all sentence case. No Title Case Except Brand Names |
| **Numbers** | Digits for 2+ | "7 phases" not "seven phases." Exception: "one" and "zero" |
| **Ampersands** | No | "and" not "&" in prose. OK in code/UI labels if space-constrained |
| **Lists** | Parallel structure | Start each item the same way (all verbs, all nouns) |
