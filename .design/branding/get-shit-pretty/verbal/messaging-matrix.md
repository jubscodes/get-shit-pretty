# Messaging Matrix

> Phase: verbal | Brand: get-shit-pretty | Generated: 2026-03-08

---

## Core Message

"Your product deserves a brand. GSP is the design journey that builds one."

This is the anchor. Every audience-specific message is a variation of this truth.

---

## Audience Messages

| Audience | Pain Point | Key Message | Tone Shift | Proof Point | Priority Channel |
|----------|-----------|-------------|-----------|------------|-----------------|
| **Developers** | Shipped code that works but looks/feels incoherent. No design process. Figma feels foreign | "You shipped the code. Now ship the brand." | More technical, slightly more casual. Speak in their workflow — CLI, config, terminal | jubs.studio was built entirely with GSP. From brief to shipped brand, in the terminal | README, GitHub, dev Twitter/Bluesky, Hacker News |
| **Designers** | Frustrated by handoff culture. Want to build, not just mock up. Feel excluded from CLI tooling | "Stop handing off. Start building." | More craft-language, slightly warmer. Reference design process they know, show it maps to GSP phases | The dual-diamond is a design framework adapted for terminal-native execution. Same rigor, new medium | Design Twitter/Bluesky, Figma community, portfolio sites |
| **Design Engineers** | Exist in the overlap but few tools validate the hybrid. Constantly context-switching between design and code | "Finally, a tool that validates the hybrid." | Balanced. These builders speak both languages fluently. Be precise, be direct, skip the onramp | GSP's pipeline covers strategy through system — the full scope a design engineer actually owns | Personal sites, niche communities, word of mouth, conference talks |
| **All Builders** | Products ship fast but don't hold together. Brand coherence is nobody's job | "Every product ships with brand coherence, not just code quality." | Manifesto mode. Declarative. This is the broad positioning for landing pages and announcements | "Three brands, one maker. Same dark-first DNA. Three distinct identities." The jubscodes ecosystem proves it works | Landing page, launch posts, README hero |

## Message Hierarchy

Use this to decide what to lead with:

1. **Hero / first impression:** Core message or all-builders message (manifesto mode)
2. **Segment landing:** Audience-specific key message (mentor mode)
3. **Feature context:** Connect capability to the audience's pain point
4. **Proof:** jubs.studio as case study. Ecosystem as evidence of range

---

## Ecosystem Messaging

When and how to reference the broader jubscodes ecosystem:

| Context | Reference | Framing |
|---------|-----------|---------|
| **Proof of concept** | jubs.studio | "Built with GSP. The tool designed the designer." |
| **Ecosystem breadth** | All three brands | "Three brands, one maker. Same dark-first DNA. Three distinct identities." |
| **Component story** | cyphercn | "cyphercn handles the components. GSP handles the brand they live inside." |
| **Attribution** | jubscodes | Always "by jubscodes" — not "a jubs.studio product" or "from jubs" |

### When NOT to reference the ecosystem

- First contact with GSP. Lead with GSP's own value. The ecosystem is a proof point, not the pitch.
- In documentation. Docs are about the tool, not the maker.
- When it would dilute the message. One brand per sentence.
