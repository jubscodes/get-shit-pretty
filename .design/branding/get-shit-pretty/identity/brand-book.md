## 20-Page Brand Book Outline

### Page 1: Cover

True black. The sparkle onboarding art centered: `░▒▓█ GET SHIT PRETTY █▓▒░` with scattered stars. Living diamonds at bottom: `◇◇`. JetBrains Mono. Nothing else. The void is the design.

### Page 2: Manifesto

Full-page typographic spread. "GSD gets shit done. GSP gets shit pretty." JetBrains Mono Display XL. Burnt orange on black. One page, one statement.

### Page 3: The Story

Why GSP exists. The gap between "it works" and "it holds together." Vibe coding backlash. The terminal as design medium. Creator + Magician archetype narrative. Inter body text, JetBrains Mono pull quotes.

### Page 4: Brand Prism

Kapferer hexagon rendered in box-drawing characters. Six facets with one-line summaries. The strategic foundation that drives every visual decision.

### Page 5: The Living Mark

Logo system overview. The concept: the logo IS the interface. Full lockup `/gsp: ◇◇` with diamond state progression. ASCII block letter banner. The install sparkle art as the brand's signature moment.

### Page 6: Logo Variations

All variations at scale: full lockup, symbol only, wordmark only, extended form, ASCII fallback. Clear space diagrams. Minimum sizes. Monochrome treatments.

### Page 7: Logo Usage Rules

Dos and don'ts. Correct rendering in JetBrains Mono. Diamond states must reflect real state. No effects, no modifications. Misuse examples with strike-throughs.

### Page 8: Color System -- Primary

Burnt orange deep dive. Hex, RGB, HSL, OKLCH, Pantone, CMYK. The 11-stop OKLCH scale from tints.dev. Strategic rationale: ecosystem inheritance from jubs.studio, complementary to cyphercn, citric warmth against void.

### Page 9: Color System -- Surfaces and Text

Five-level surface hierarchy from #000000 to #282828. Three-level text hierarchy. Border system with accent. All combinations shown with WCAG contrast ratios.

### Page 10: Color System -- Semantic and Ecosystem

Success, warning, error, info with rationale. Ecosystem color inheritance table: jubs.studio, GSP, cyphercn. How the three brands read as family.

### Page 11: Typography -- JetBrains Mono

Specimen page. Full character set. Weight range (Regular through ExtraBold). Use cases. Why monospace IS the identity, not a secondary treatment. Ecosystem comparison with jubs.studio and cyphercn.

### Page 12: Typography -- Inter

Specimen page. Full character set. Weight range (Regular through Bold). The warmth complement. How it maps to the "warm (not soft)" voice attribute.

### Page 13: Type Scale and Hierarchy

Full scale table from Display XL (72px) through Code SM (12px). Responsive behavior. Pairing rules. Example compositions showing JetBrains Mono headings with Inter body.

### Page 14: Imagery -- Screenshots and ASCII

Screenshot treatment guidelines. Terminal captures as hero images. The install sparkle art and ASCII block banner as signature visual artifacts. Before/after transformation compositions.

### Page 15: Imagery -- Diagrams and Texture

Diagram style (monochrome + orange accent, node-and-edge, flowchart). Dot grid pattern specs. Syntax highlighting as texture. Box-drawing characters as structural elements.

### Page 16: Visual Style -- Neubrutalism

The dark neubrutalist system. Thick borders, flat surfaces, no gradients, no glassmorphism. How this differs from Linear-derivative dark themes. The avant-garde position.

### Page 17: Voice and Tone

Brand voice summary (precise, irreverent, warm, craft-forward). Tone spectrum visualization. Warm mentor default, manifesto for key moments. Copy examples across touchpoints.

### Page 18: Applications -- Terminal and CLI

Pipeline progress with living diamonds. Phase completion trees. Error states. Statusline. The install experience as brand storytelling. How every CLI interaction is a brand moment.

### Page 19: Applications -- Web and Social

Landing page layout. README structure. GitHub/Twitter presence. Documentation treatment. npm package page.

### Page 20: The Ecosystem

jubs.studio + GSP + cyphercn. Shared DNA, distinct energy. Color inheritance diagram. Type relationship map. The proof point: jubs.studio was built with GSP. The tool designed the designer.
