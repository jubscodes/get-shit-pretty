# Brand Book

> Phase: identity | Brand: get-shit-pretty | Generated: 2026-03-19

---

## 20-Page Brand Book Outline

### Section 1: Brand Story (Pages 1-3)

| Page | Content |
|------|---------|
| 1 | **Cover.** Void black background. `/gsp` mark in amber, centered. `get-shit-pretty` in Instrument Sans below. No tagline on cover -- the name says enough. |
| 2 | **Purpose.** "AI made building fast. Nothing made building coherent." Full-page statement in Instrument Sans 700. The problem we exist to solve, in one breath. |
| 3 | **Story.** Origin narrative -- why a design process for the terminal. Vision, mission, values as a structured layout. The dual-diamond diagram showing brand and project journeys. |

### Section 2: Strategy Foundation (Pages 4-5)

| Page | Content |
|------|---------|
| 4 | **Archetype.** Creator + Guide explained visually. Two-column layout: Creator traits (precision, systems, craft) left, Guide traits (warmth, explanation, alongside) right. How they blend in practice. |
| 5 | **Positioning.** The positioning map (process vs. pieces, designer-native vs. developer-native). GSP's white space claim. Competitive context without competitor logos. |

### Section 3: Voice & Tone (Pages 6-7)

| Page | Content |
|------|---------|
| 6 | **Voice attributes.** Clear, Warm, Lighthearted -- each with do/don't examples. The tone spectrum visual showing context shifts. |
| 7 | **Writing samples.** Real GSP output demonstrating voice in action: CLI output, error messages, completion messages, README excerpt, social post. Annotated with attribute labels. |

### Section 4: Logo System (Pages 8-10)

| Page | Content |
|------|---------|
| 8 | **Primary mark.** The Slash Mark (`/gsp`) at scale. Construction grid showing character relationships. Clear space diagram. |
| 9 | **Variations.** All four variations (primary, inline, icon, wordmark) shown on dark and light backgrounds. Minimum size reference. ASCII rendering in actual terminal screenshot. |
| 10 | **Usage rules.** Correct and incorrect usage. Color applications (amber slash, monochrome, reversed). The supporting diamond system for phase indication. |

### Section 5: Color (Pages 11-13)

| Page | Content |
|------|---------|
| 11 | **Palette overview.** Full foundation palette (void through bright) + amber accent. Large swatches with hex values. The 70/25/5 ratio visualized. |
| 12 | **Color in context.** Terminal output showing ANSI color rendering. Web application showing the palette in use. Dark-to-light mode mapping. |
| 13 | **Semantic colors and rules.** Error, success, warning, info with use cases. Contrast ratio table. Color don'ts (no gradients, no decorative amber, no new accent colors). |

### Section 6: Typography (Pages 14-15)

| Page | Content |
|------|---------|
| 14 | **Typeface showcase.** JetBrains Mono character set at multiple weights. Instrument Sans display specimens. Pairing rationale connected to voice attributes. |
| 15 | **Type scale and rules.** Full scale table from Display 1 to Overline. Terminal vs. web hierarchy. Responsive behavior. Usage rules and pairing logic. |

### Section 7: Imagery & Texture (Pages 16-17)

| Page | Content |
|------|---------|
| 16 | **ASCII art direction.** Character vocabulary table. Composition examples: phase diagrams, progress bars, structural art. Whitespace principles. |
| 17 | **Screenshots and texture.** Terminal screenshot styling guide. Dot grid specification. Photography and illustration policies (none -- with rationale). |

### Section 8: Layout & Structure (Page 18)

| Page | Content |
|------|---------|
| 18 | **Grid and spacing.** 8px base grid visualized. Terminal column grid (80-char). Border radius (2-4px), border weight (1px), shadow policy (none). Motion specification (100-150ms, linear). |

### Section 9: Applications (Page 19)

| Page | Content |
|------|---------|
| 19 | **Brand in context.** Six touchpoints shown at actual size: terminal output, GitHub README, npm page, social card, conference slide, documentation page. Each annotated with key design decisions. |

### Section 10: Do's and Don'ts (Page 20)

| Page | Content |
|------|---------|
| 20 | **Quick reference.** Two-column do/don't grid covering logo, color, typography, imagery, and voice. Each item is a specific, actionable rule -- not a vague principle. Closes with: "The brand is the proof. Every detail matters." |
